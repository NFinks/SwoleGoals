Data Scraping:

bodybuilding.com
strengthlevel.com

Web scraping was done by utilizing Selenium web browser automation. A Python script was written which loops through each of the muscle groups found here (17 total), selecting the “Load More” button once on each muscle group, and getting the URLs for each exercise that is present on that page by using its xpath. These URLs are then loaded in the browser, and the exercise name, exercise type, muscle worked, equipment needed, exercise ranking, exercise instructions, and both exercise images are located using their xpaths and stored into arrays. The totality of the data is then written to a tab separated file that then can easily be put into the MySQL database. strengthlevel.com was scraped similar to bodybuilding.com, but we decided to scrape by modifying two select tags to parse through all available exercises and obtain performance information based on a 150 pound individual.

Tools

A MySQL database running on Google Cloud contains all of the exercise information.
Express is used to setup routes to get, post, delete, and update data from the database
Our frontend is built using Angular running on Google Cloud
NodeJS is used to build the backend server
AWS is used to host the frontend Angular project
All of our code is hosted and revision controlled in our Github repository here
Bootstrap provides us with prebuilt CSS classes and a grid system to make laying out the frontend easier
We are using Google App Engine for our Firestore and SQL backend servers
We use npm to download and manage dependencies like Bootstrap
We use Firestore to host our non-relational data such as user data, groups, and challenges
Travis CI runs our tests on a VM and then deploys to AWS after the master branch passes our tests
We are using Python to scrape data
We use Selenium to get the exercise information from bodybuilding.com
We are using Vim, Visual Studio Code, and Jetbrains Webstorm to edit our code
We are using Postman to test our backend API. We exported our Postman collection and are running the tests using Travis CI and Postman's Newman CLI.
