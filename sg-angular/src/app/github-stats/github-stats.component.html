Data Scraping

bodybuilding.com

Web scraping was done by utilizing Selenium web browser automation. A python script was written which loops through each of the muscle groups found on https://www.bodybuilding.com/exercises/finder/? (17 total), selecting the “Load More” button once on each muscle group, and getting the URLs for each exercise that is present on that page by using its xpath. These URLs are then loaded in the browser, and the exercise name, exercise type, muscle worked, equipment needed, exercise ranking, exercise instructions, and both exercise images are located using their xpaths and stored into arrays. The totality of the data is then written to a tab separated file that then can easily be put into the MySQL database.

Tools

A MySQL database running on Google Cloud contains all of the exercise information.
Express is used to setup routes to get, post, delete, and update data from the database
Our frontend is built using Angular running on Google Cloud
NodeJS is used to build the backend server
All of our code is hosted and revision controlled in our Github repository here
Bootstrap provides us with prebuilt CSS classes and a grid system to make laying out the frontend easier
We are using three separate Google Cloud servers, one to host the database, one to host the backend, and one to host the frontend
We use npm to download and manage dependencies like Bootstrap
We are using Python to scrape data
We use Selenium to get the exercise information from bodybuilding.com
We are using Vim, Visual Studio Code, and Jetbrains Webstorm to edit our code
We are using Mocha to create unit tests
